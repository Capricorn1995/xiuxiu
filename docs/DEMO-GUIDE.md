# 投资人演示指南

本文档为「嗅嗅」面向投资人的功能演示脚本，覆盖核心使用场景。建议演示时长 5–8 分钟，演示前请完成「演示准备」中的所有事项。

## 演示准备

### 环境清单

- Windows 10/11 笔记本，已安装「嗅嗅」v0.1.0
- 稳定网络（建议有线或 5GHz Wi-Fi）
- 外接显示器或投屏（分辨率 1920×1080 及以上）
- 提前关闭通知、IM 提醒，避免演示被打断

### 演示数据准备

为避免现场依赖外部网络稳定性，建议提前准备以下页面：

| 演示场景 | 推荐页面 | 备注 |
| :--- | :--- | :--- |
| 图片嗅探 | 任意电商商品详情页 / 图虫摄影首页 | 图片密集，效果直观 |
| 视频嗅探 | 任意新闻网站视频页 / 短视频平台 | MP4 直链为主 |
| M3U8 下载 | 任意 HLS 直播回放页 / 在线课程页 | 现场嗅探 M3U8 链接 |
| 批量下载 | 图墙类页面（如 Unsplash 探索页） | 一次触发 10+ 资源 |

> 准备完成后，建议完整走通一遍演示流程，预估每个环节耗时。

### 演示目标

在 5–8 分钟内让投资人清晰理解三件事：

1. **产品价值**：解决什么问题（网页资源获取门槛高）
2. **技术壁垒**：CDP 嗅探 + 内置 ffmpeg 合并，体验远超浏览器插件
3. **商业潜力**：用户基数、付费意愿、扩展空间

## 演示步骤详解

### 场景一：图片嗅探演示

**演示话术**：

> 「传统方式保存网页图片，需要逐张右键另存，效率低下。我们看嗅嗅是怎么做的。」

**操作步骤**：

1. 启动「嗅嗅」，在内置浏览器地址栏输入预先准备好的图片密集页 URL
2. 页面加载过程中，右侧「嗅探列表」实时滚动出现图片资源缩略图
3. 点击列表上方的「类型筛选 → 图片」，仅展示图片资源
4. 选中任意一张图片，点击「下载」按钮
5. 弹出文件保存对话框（或按默认路径直接下载），完成后在「下载管理」中查看

**关键展示点**：

- 实时性：页面加载即可见，无需等待
- 缩略图预览：所见即所得
- 元信息展示：分辨率、大小、域名、MIME 类型
- 速度：从触发到完成通常 < 2 秒

### 场景二：视频嗅探演示

**演示话术**：

> 「很多视频网站不提供下载入口，甚至右键菜单被禁用。嗅嗅能直接捕获视频真实地址。」

**操作步骤**：

1. 在「嗅嗅」地址栏输入预先准备的视频页 URL
2. 播放页面中的视频
3. 在「嗅探列表」中切换到「视频」筛选标签
4. 展示捕获到的 MP4 直链（高亮显示 MIME 类型为 `video/mp4`）
5. 点击「下载」，观察下载进度

**关键展示点**：

- 捕获到的是原始视频文件 URL，而非播放器封装
- 支持识别分辨率、时长（如可解析）
- 对比传统方式（F12 查找视频地址）的体验差异

### 场景三：M3U8 流媒体下载演示

**演示话术**：

> 「HLS 流媒体是当前主流的视频分发方案，传统工具下载 M3U8 需要命令行 ffmpeg，普通用户难以使用。嗅嗅把它做成了一键操作。」

**操作步骤**：

1. 在「嗅嗅」地址栏输入预先准备的 HLS 视频页 URL
2. 播放视频，嗅探器自动捕获 `.m3u8` 请求
3. 在「嗅探列表」中找到 M3U8 资源，点击「下载」
4. 弹出码率选择对话框（如为 master playlist），选择最高码率
5. 观察下载流程：
   - 解析播放列表
   - 分片下载（展示分片进度条）
   - 调用内置 ffmpeg 合并为 MP4
6. 下载完成后，直接在「下载管理」中点击「打开文件」用系统播放器播放

**关键展示点**：

- 全程零命令行，普通用户也能完成
- 自动识别 master/media playlist，支持码率选择
- 内置 ffmpeg，无需用户额外安装
- 合并后的 MP4 文件可在任意播放器播放
- **技术壁垒**：自研分片调度 + 合并流程，对加密 HLS（AES-128）也有支持

### 场景四：批量下载演示

**演示话术**：

> 「当你需要保存一组图片或一个图集时，逐张下载显然不现实。嗅嗅支持批量选择与并发下载。」

**操作步骤**：

1. 打开图墙类页面（如 Unsplash 探索页）
2. 等待嗅探列表捕获 10+ 张图片
3. 点击列表顶部的「全选」或勾选多张图片
4. 点击「批量下载」
5. 切换到「下载管理」标签页，观察：
   - 多个任务并发执行（默认 3 路并发）
   - 实时显示每个任务的速度与进度
   - 支持单个任务暂停/恢复
6. 等待全部完成，展示「已完成」列表

**关键展示点**：

- 并发控制：可在设置中调整最大并发数
- 失败重试：单个分片失败自动重试
- 智能命名：批量下载时按 `{title}_{index}` 自动命名，避免冲突
- 后台运行：最小化到托盘后继续下载

## 演示技巧

1. **节奏控制**：每个场景 60–90 秒，避免在单个细节上停留过久
2. **预置资源**：演示前确保所有页面已加载过一次，避免现场卡顿
3. **应对意外**：若现场网络异常，立即切换到本地 HTML 演示页（建议提前准备）
4. **强调差异**：每个场景后用一句话总结「这比传统方式快/简单多少」
5. **数据收尾**：最后用数据指标收束，从「产品体验」过渡到「商业价值」

## 常见提问预案

**Q：和浏览器插件（如 Video DownloadHelper）相比有什么优势？**
A：插件受浏览器权限限制，无法访问跨域请求与加密流；嗅嗅基于系统级 CDP，能力等同于开发者工具，且内置 ffmpeg 完成合并，体验远超插件。

**Q：会被网站反爬机制封禁吗？**
A：嗅探是被动监听，不发起额外请求，对目标站点零压力；下载部分遵循标准 HTTP 协议，与浏览器行为一致。

**Q：商业模式？**
A：工具基础功能免费，Pro 订阅提供云转码、跨设备同步、批量任务队列；企业版提供 API 与团队管理。

**Q：竞争壁垒？**
A：CDP 嗅探引擎 + M3U8 解析合并流水线有较高工程门槛，配合本地 ffmpeg 调度优化，体验差距在 6 个月内难以被复制。
